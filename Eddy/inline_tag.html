Inline elements

Why do we bother with inline elements? There are certainly times we want to mark up content, but we don't want to insertthe line breaks around them. The < span> tag was the first to gie you that ability, but it was mostly used to style the code, and we don't want to do that. Some of the newer tags that still seperate content and style are listed below. Remember, inline tags can flow next to each other and don't cause a line break before and after the element. We have seen this with links and images.

Citing a source - cite

The < cite> makes it much easier for people and bots to search the code for this specific information.

A great reference for these and other elements is https://developer.mozilla.org/en-US/docs/Web/HTML

Defining your abbresviations - abbr

The < abbr> will display an abbreviaitons on the browser. But if a title attribute is special in the tag, the title will appear when the mouse hovers over the abbreviation. Screen readers also provide the title information.

The days of the week are often abbreviated: Sun, Mon, Tues, etc.

Specifying the use of time - time

The < time> is again a completely sementic tag that can be used to search for specific information.

The party starts at 9:00pm.

The party took place on New Year's Eve.

Identifying code fragments - code

The < code> element tends to be displayed in a different font, but also has the semantic meaning to convey information to as many as possible.

Using subscripts and superscripts - sub and sup

The < sub> and < sup> elements are used to create the super and subscirpts, e.g.(x+y)3 or H2O. In the past, people would often paste mathematical images into the page. But images didn't convey the same meaning to all users.
